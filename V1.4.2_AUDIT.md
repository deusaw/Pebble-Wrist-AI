# Wrist AI v1.4.2 Audit

Audit date: 2026-07-01
Scope: watch C, PebbleKit JS, Config, message protocol, storage, Timeline, ToDo & Notes, Wakeup, TTS, documentation and five-platform build.

## Result

No release-blocking defect was found by static analysis or automated integration checks. The project builds a five-platform PBW successfully.

The remaining risks require physical-device or companion-app validation and must not be represented as verified:

- Config export must still be tested from the iOS Pebble App with Chinese, emoji and a maximum-length conversation.
- ToDo & Notes list navigation, conversation restoration, seven reminder slots and the
  reserved Todoist sync Wakeup require real-watch testing.
- Flint/Duo TTS remains experimental because no Duo hardware is available.
- Core Devices currently ignores third-party Timeline developer actions, so `openWatchApp` is future-ready but not user-visible today.
- Core Devices' `insertTimelinePin` and `deleteTimelinePin` bridges have no
  completion callbacks. Wrist AI can confirm submission, not watch delivery.
- iOS may delay or fail a scheduled network sync after a phone reboot until the
  Pebble App has been opened once.

## Corrections included in the audited tree

- Fixed the Notes Complete/Reopen/Delete crash path: the watch no longer mutates
  or reloads the underlying Notes `MenuLayer` from the action flow. The phone is
  the single source of truth and returns a full `NOTE_LIST` snapshot 600 ms
  later, after the action window has closed.
- Before parsing a snapshot whose row count may shrink, the watch moves the
  Pebble SDK's selection to row zero, reloads the new data, then restores a
  clamped selection. This prevents the old last-row index from entering
  `draw_row` after a deletion.
- Note mutations are now queued in watch persistent storage from the menu
  callback and transmitted 900 ms later when the UI is idle. Failed operations
  remain queued, retry automatically, and survive a watch-app restart.
- Note commands are idempotent (`complete`, `reopen`, `delete`) so a retry
  cannot accidentally toggle a completed item back to open.
- Completing or deleting an ordinary Note no longer sends an unnecessary
  `NOTE_WAKEUP cancel` message. Wakeup cancellation is limited to Notes that
  actually have a strong scheduled reminder.
- Reopening an ordinary Note follows the same rule. Previously its separate
  rescheduling branch still sent `NOTE_WAKEUP cancel` unconditionally; this was
  the remaining crash path after Complete/Delete had been stabilized.
- Fixed timed Timeline requests such as `提醒我明天下午3点开会`. The validator now
  accepts `relative_days + local_time`, `date + local_time`, clock-only `time`
  combined with a date, and numeric `hour/minute`.
- Added a deterministic Chinese fallback that recovers today/tomorrow/day-after
  plus a clock time from the original user request if a planner omits the event
  time. Absolute ISO times without an offset remain interpreted in the phone's
  local timezone.
- Updated the Timeline planner schema with an explicit
  `relative_days: 1, local_time: "15:00"` example.
- Untimed enumerated reminders are treated as TODO Notes, not Timeline events.
  The exact request `三个提醒“洗衣服、买菜、做饭”` deterministically creates
  three Notes even if both LLM control-block paths fail. The fallback refuses
  requests containing a date or time, preventing duplicate Timeline/Note items.
- Moved the large Emery/Flint TTS PCM ring to heap allocation, removing the Emery 65,535-byte static-image failure while preserving playback capacity.
- Prevented opening a Note conversation from popping the main application window.
- Prevented a failed Timeline insertion from leaving an orphan linked ToDo & Notes item.
- Re-schedules strong reminders when a TODO is reopened or its time changes.
- Uses an all-day event's explicit reminder time for a linked strong Wakeup instead of midnight.
- Limits full active-Note context to its originating chat, preventing Note content from leaking into unrelated conversations.
- Restores a Note's original chat when Wrist AI is launched by its Wakeup; creates a replacement chat only if the original was deleted.
- Reorganized Config without changing existing control IDs or the PebbleKit configuration response schema.
- Updated the capability prompt so Wrist AI mentions ToDo & Notes when asked what it can do.
- Added serverless Todoist two-way synchronization with a durable offline
  outbox, stable Todoist task IDs, incremental Sync API reads, project scoping,
  conflict-safe idempotent completion/reopening and Homepage status.
- Todoist due dates project to Timeline; adding or changing due upserts a Pin,
  while removing due, completion and deletion use Core Devices'
  `Pebble.deleteTimelinePin`. Reopening receives a new Pin ID because Pebble
  forbids reuse after deletion.
- Added a reserved rolling sync Wakeup. Seven slots remain for strong reminders;
  a one-minute Wakeup collision shifts synchronization by two minutes.
- Config now includes Todoist enablement, token lifecycle, project selection,
  connection testing and optional scheduled synchronization.

### iOS Settings loader correction

The first v1.4.2 test PBW copied the source file directly and left
`require('opencc-js/t2cn')` unresolved. The watch binary could install, but
PebbleKit JS stopped before registering `showConfiguration`, so iOS could not
open Settings. The legacy single-file build now vendors the OpenCC UMD bundle
into the root `pebble-js-app.js` without enabling MultiJS. The final PBW is
unpacked and executed without external `require`; the test verifies Config URL
creation and a real Traditional-to-Simplified conversion.

### ToDo & Notes / Timeline source-of-truth audit

- Renamed the user-facing local list to **ToDo & Notes** on watch, Config and
  docs. Internal protocol names remain `NOTE_*` for compatibility.
- Wrist AI-created Timeline pins are now always represented by a local
  ToDo & Notes item. Pure appointments become `event` items; actionable items
  can still be `todo`.
- ToDo & Notes items with `timeline_pin_id` delete the local Timeline pin via
  `Pebble.deleteTimelinePin` when completed/deleted, and Todoist task IDs are
  linked back to both the local item and the Timeline map.
- Todoist due tasks import as ToDo & Notes `event` items and are projected to
  Timeline. Untimed Todoist tasks import as TODO items.
- Existing Timeline map entries without a local item are reconciled into
  ToDo & Notes on the next list refresh.
- Core Devices exposes insert/delete bridges but no PKJS Timeline list/read API,
  so Wrist AI can only manage pins it created and saved locally.
- Latest test PBW: `build/Wrist-AI-v1.4.2-todo-notes-timeline-source.pbw`
  SHA-256 `c5eda185d31ab9c928338ef4bead84ad6d02e01575f70e3f72cb9a5585f5daf9`.

### Todoist post-sync crash correction

- Reproduced the control-flow distinction for an empty Todoist account and an
  empty local item store: the Note list and `Synced` status are both handled
  during startup, while the rolling OS Wakeup was first applied after the
  network sync completed.
- Wakeup cancellation/query/scheduling is no longer executed inside the
  AppMessage inbox callback. It is deferred to an app timer, coalesces repeated
  requests, and preserves an existing Wakeup when the requested time is within
  90 seconds.
- The timer is cancelled during app teardown, preventing a callback from
  touching deinitialized app state.
- Five-platform compilation, JavaScript syntax validation, PBW archive
  validation and `git diff --check` pass.
- Test PBW: `build/Wrist-AI-v1.4.2-sync-crashfix.pbw`
  SHA-256 `498d4c73820f948991daaea86c38bf727a8a14c3b1eb7fdd3d94db9ddac39104`.
- Flint real-device logging subsequently identified the exact remaining fault:
  every launch terminated after about five seconds at `PC=0x648c`, which
  resolves to newlib `strtol`.
- All watch-side decimal control parsing now uses a bounded int32 parser instead
  of `strtol`/`atoi`. This covers sync Wakeups, Note Wakeups and model indices,
  rejects malformed/overflowing values, and removes about 1.1 KB from Flint RAM
  footprint.
- The corrected PBW was installed over Developer Connection and remained open
  beyond the former deterministic crash window with no further app fault.

## Storage and limits

| Data | Location | Limit |
|---|---|---:|
| Conversations | Phone `localStorage` | 20 chats |
| Messages | Phone `localStorage` | 20 messages per chat |
| Memory.md | Phone `localStorage` | 40 items / 12 KB |
| ToDo & Notes | Phone `localStorage` | 50 items |
| Watch Note list | AppMessage / watch heap | 20 items |
| Strong reminders | Watch persistence / Wakeup API | 7 slots |
| Todoist sync Wakeup | Watch persistence / Wakeup API | 1 rolling slot |
| Todoist offline outbox | Phone `localStorage` | Durable / compacted |
| Todoist incremental token | Phone `localStorage` | 1 |
| Visible reply | Phone truncation / watch buffer | 1,800 UTF-8 bytes |
| Timeline launch map | Phone `localStorage` | 40 entries |

## Release decision

The code is suitable for a v1.4.2 test build. Public release should wait for the real-device checks above, especially iOS Config export, Emery TTS regression, and Notes/Wakeup navigation.
